# Unidad 2 - Sesión 5

## Sesión 10 - Dashboard BI con KPIs y visualización base

Este documento deja congelada la quinta sesión macro de la Unidad 2.

La sesión corresponde al bloque:

```text
Unidad 2 - Construcción del BI
Sesión 10 - Dashboard BI: KPIs, visuales principales, filtros y lectura ejecutiva
```

## Por qué esta sesión va después de la exploración

Una vez que el estudiante ya sabe navegar jerarquías, abrir detalles y controlar contexto de filtro, puede diseñar una página de reporte con intención.

Aquí el foco cambia:

```text
Explorar el modelo -> Seleccionar indicadores -> Diseñar una página ejecutiva
```

El dashboard no se arma como una colección de gráficos. Se arma como una respuesta visual a preguntas de negocio.

## Alcance

- tarjetas KPI
- indicadores comerciales base
- KPI comparativo de variación
- diseño de una página `Resumen BI`
- gráfico temporal de ventas
- matriz o visual jerárquico por producto
- rankings de clientes y vendedores
- tabla KPI con ventas actuales, ventas año previo, variación e iconos
- filtros visibles para usuario final
- formatos de moneda, porcentaje y conteos
- validación de KPIs contra SQL

## Lógica didáctica

- el estudiante convierte el análisis exploratorio en una página ejecutiva
- define qué métricas merecen ser KPI
- diferencia visuales de resumen, comparación, tendencia y detalle
- diseña filtros que el usuario final sí necesita manipular
- usa las páginas de detalle y tooltips de la sesión 4 como soporte, no como adorno
- valida ventas, pedidos, ticket y variación contra SQL

## Resultado esperado

Al finalizar la sesión, el alumno debe tener una página `Resumen BI` con:

- KPIs base de ventas, pedidos, unidades y ticket
- gráfico temporal de ventas
- matriz o visual jerárquico de producto
- ranking de clientes
- ranking de vendedores
- tabla KPI comparativa con variación e iconos
- segmentadores funcionales
- consistencia visual y formatos correctos
